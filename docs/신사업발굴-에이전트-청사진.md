# 신사업 발굴 에이전트 — 설계 청사진 (Phase 0)

> **문서 성격**: Claude Code 기반 신사업 업무 에이전트화의 방법론·아키텍처 설계 문서.
> 내부 기밀 데이터는 포함하지 않으며, 모든 예시는 공개 정보 기반이다.
> **⚠️ 기밀성**: 향후 에이전트가 생성하는 실제 리서치 산출물은 **사내 Notion(비공개)** 에만 저장하며,
> 공개 GitHub Pages 사이트에는 절대 게시하지 않는다.

---

## 1. 배경과 목표 (Why)

신사업추진파트의 신사업 업무를 Claude Code 기반 에이전트로 전환하는 것이 목표다.
신사업 업무는 다음 두 영역으로 정의된다.

| 영역 | 내용 | 에이전트화 우선순위 |
|------|------|:---:|
| **① 신사업 아이템 발굴** | 시장조사 · 경쟁환경 · 규제 조사 → **진입 초기 의견** 수립 | **1순위** |
| **② 신사업 추진방향 도출** | 진입영역 정의, Build/Borrow/Buy 비교·도출, Phase별 목표 | 2순위 |

**1번을 먼저 에이전트화하는 이유**: 1번은 업무의 대부분이 "외부 정보 수집 → 구조화 →
1차 의견 수립"으로 구성된다. 이는 LLM 리서치 에이전트가 가장 잘하는 작업이며, 자동화 시
리서치 리드타임을 크게 단축하고 분석 품질의 일관성을 확보할 수 있다. 반면 2번(특히 Buy의
Valuation·투심위)은 내부 데이터·재무모델·의사결정 거버넌스가 얽혀 있어 후속 단계가 적합하다.

1번의 최종 산출물인 **진입 초기의견서**가 2번(B/B/B)의 입력으로 연결되는 파이프라인으로 설계한다.

---

## 2. 실현 가능성 (환경 점검 결과)

본 청사진 작성에 앞서 실제 환경에서 핵심 전제를 검증했다.

- **웹 리서치 동작 확인됨.** WebSearch/WebFetch로 한국 1차 출처(보험연구원 KIRI,
  생명보험협회 KLIA, 하나금융연구소, Mordor Intelligence 등)가 정상 수집된다.
- **`deep-research` 스킬 재사용 가능.** 팬아웃 검색 → 출처 적대적 검증 → 인용 리포트 생성
  하니스가 이미 탑재돼 있어, 각 도메인 리서치의 엔진으로 활용할 수 있다.
- **연결된 MCP 서버**: Notion(산출물 저장), KakaoTalk(완료 알림), GitHub.
- 레포지토리는 사실상 백지 상태(기본 Jekyll 사이트)로 기존 `.claude/`·`CLAUDE.md`가 없어
  자유롭게 설계할 수 있다.

## 4. ① 발굴 에이전트 — 아키텍처

### 4.1 구조: 오케스트레이터 + 도메인 서브에이전트(병렬) + 종합

```
/신사업발굴 "AI 웰니스로봇"
        │
        ▼
┌───────────────────────────┐
│  오케스트레이터            │   ← 스킬 new-business-discovery
│  (스코프 정의 → 분배 → 종합)│
└───────────────────────────┘
        │  병렬 dispatch
        ├──────────────┬──────────────┬──────────────┐
        ▼              ▼              ▼              │
  ┌──────────┐  ┌──────────┐  ┌──────────┐          │
  │ 시장조사  │  │ 경쟁환경  │  │  규제     │          │
  │ market-  │  │competitive│ │regulation-│          │
  │ research │  │-landscape │ │-insurance │          │
  └────┬─────┘  └────┬─────┘  └────┬─────┘          │
       │ 리포트      │ 리포트       │ 리포트          │
       └─────────────┴──────────────┴────────────────┘
                          ▼
                  ┌──────────────┐
                  │ 진입 초기의견  │  entry-opinion
                  │ (종합·권고)    │
                  └──────┬───────┘
                          ▼
            Notion 저장 ── (옵션) KakaoTalk 알림
```

### 4.2 진입점 — 슬래시 커맨드
- `/신사업발굴 <산업/영역>`  예) `/신사업발굴 AI 웰니스로봇`
- 입력은 산업/사업 영역. 필요시 옵션(지역, 분석 깊이, 비교 대상 등)을 인자로 받는다.

### 4.3 오케스트레이터 (스킬 `new-business-discovery`)
1. 입력 영역을 받아 **리서치 스코프**를 정의(핵심 질문 도출, 분석 경계 설정).
2. 3개 도메인 서브에이전트를 **병렬로 dispatch**(격리된 컨텍스트에서 동시 실행).
3. 3개 리포트를 모아 **진입 초기의견 에이전트**로 종합.
4. 결과를 **Notion**에 저장하고, (옵션) **KakaoTalk**으로 완료 알림.

### 4.4 도메인 서브에이전트 3종 (병렬, 각자 표준 리포트 1건)

| 에이전트 | 다루는 것 | 주요 출처 |
|----------|-----------|-----------|
| **시장조사** `market-research` | 시장규모·성장률·성장동인, TAM/SAM/SOM, 세그먼트, 수요 트렌드 | 통계청, KIRI, 증권사 리포트, 글로벌 리서치(Mordor 등), 업계 협회 |
| **경쟁환경** `competitive-landscape` | 주요 플레이어 맵·BM·점유율, 경쟁 다이내믹스(5 Forces), 향후 시나리오 | DART 공시, IR자료, 뉴스, 기업 홈페이지 |
| **규제** `regulation-insurance` | 업계 규제 + **생보사 진입요건** | 금융위, 금감원, 국가법령정보(law.go.kr), KIRI |

**규제 에이전트의 생보사 진입요건 체크리스트(핵심)**
- 보험업법상 **부수업무·겸영업무** 신고/승인 대상 여부
- 금융위·금감원 **인허가** 필요 사항
- **자회사 출자** 규제 (보험업법 §115 — Buy/지분투자와 직접 연결)
- **건강증진형 보험상품 가이드라인**, 마이데이터, 개인정보·의료정보 규제

### 4.5 종합 — 진입 초기의견 `entry-opinion`
3개 도메인 리포트를 종합하여 다음을 산출한다.
- **시장매력도** / **경쟁강도** / **규제난이도·리스크** 평가
- **생보사 역량 적합성(Strategic Fit)** — 기존 자산·채널·고객기반과의 연계성
- **진입 권고: Go · Watch · No-go** + 근거
- **다음 단계 핸드오프**: 2번 영역(Build/Borrow/Buy)으로 넘길 초기 방향 제안

## 5. 표준 템플릿 (4종)

향후 구현 시 `.claude/skills/new-business-discovery/templates/` 에 둔다.
1. **시장조사 리포트** — 규모/성장/동인/세그먼트/트렌드
2. **경쟁환경 리포트** — 플레이어 맵/BM/점유율/다이내믹스/시나리오
3. **규제 리포트** — 업계 규제 + 생보사 진입요건 체크리스트
4. **진입 초기의견서** — 1p Exec Summary + 평가표 + 권고(Go/Watch/No-go)

## 6. 출처·신뢰성 원칙

- **한국 1차 출처 우선** — 통계청·금융위·KIRI·KLIA·DART 등.
- **모든 수치에 인용** — 연도·통화·출처를 함께 명시.
- **`deep-research`의 적대적 검증 단계** 활용으로 환각·과장 통계 방지.
- **규제 해석은 '1차 의견'** 임을 명시 — 최종 법무·컴플라이언스 검토가 반드시 필요.
- WebSearch는 US 기반·시점 한계가 있으므로, 핵심 수치는 원문 출처(WebFetch)로 교차 확인.

## 7. 워크드 예시 — AI 웰니스로봇 (SK인텔릭스 '나무엑스 A1' 모델 참고)

> 설계 검증용 시연. 실제 리서치 산출물이 아니라 "에이전트가 이렇게 흘러간다"를 보여주는 예시.

**입력**: `/신사업발굴 AI 웰니스로봇`

- **시장조사**: 글로벌/국내 웰니스 로봇·홈 헬스케어 디바이스 시장 규모·성장률, 고령화에
  따른 시니어케어 수요. → TAM/SAM/SOM 추정.
- **경쟁환경**: SK인텔릭스(나무엑스 'A1' — 자율주행 + 생체정보 측정(체온·심박·스트레스·
  맥박·산소포화도) + 공기청정 + 음성응답, CES 2026 혁신상, 출시 후 한 달 2,000대+ 판매),
  삼성/LG 가전·홈로봇, 글로벌 ElliQ 등. → 플레이어 맵·향후 시나리오.
- **규제**: 의료기기 vs 웰니스(공산품) 분류(식약처), 생체정보 처리(개인정보보호법),
  생보사 부수업무 해당 여부, 건강증진형 보험 연계 가이드라인.
- **진입 초기의견**: 건강증진형 보험·시니어케어와의 번들 적합성 평가, B/B/B 초안
  (예: **Borrow** = SK인텔릭스 등과 제휴 / **Buy** = 지분투자) → 권고 + 2번 영역 핸드오프.

## 8. 로드맵 (Phase)

| Phase | 내용 | 산출물 |
|-------|------|--------|
| **0 (이번)** | 본 청사진 확정 | 설계 문서 |
| **1** | ① 발굴 에이전트 실제 구축 — `.claude/`(skill+agents+command)+템플릿+Notion 연동, 파일럿 1건(AI 웰니스로봇) end-to-end 검증 | 동작하는 발굴 에이전트 |
| **2** | ② 추진방향 에이전트 — Build/Borrow/Buy 비교 프레임워크, Valuation 보조, 투심위 자료 초안 | 추진방향 에이전트 |
| **3** | 사내 데이터 연동(과거 신사업 검토 이력 Notion DB 참고), 정기 산업 스캐닝(스케줄링) | 상시 운영 체계 |

## 10. 리스크·주의사항

- **기밀성**: 실제 리서치 산출물은 비공개 Notion에만 저장, 공개 사이트 게시 금지.
- **수치 환각 방지**: 인용 강제 + `deep-research` 검증 단계.
- **규제는 1차 의견**: 최종 법무·컴플라이언스 검토 필수.
- **웹검색 한계**: US 기반·시점 한계 → 핵심 수치는 원문 교차 확인.
